# 创作DNA工坊 · 产品需求文档(PRD)

> 状态:定稿(grill-me 重定义 2026-06-03)。本地文档,未发布追踪器。
> 配套文档(均为本 PRD 的组成部分):
> - **UX 唯一真相源:** `docs/redesign-mockup.html`(5 屏高保真原型)
> - **引擎/实现蓝图:** `docs/redesign-engine-blueprint.md`(逐文件对账真实代码)

## 1. 一句话定位

**选两本写完的书——一本当骨架、一本换题材——一键生成一本"形似神不似"的新书开篇。**

## 2. 为什么重做

现状把整条工业流水线(近 30 个手动旋钮:切分校验、档位、100/全量、偏向滑块、温度、红队框、分镜数、命令栏微调……)直接摊给用户,导致"**用不明白、太复杂、手动太多、缺乏业务逻辑**"。本次重定义的总动作:**做减法 + 把决策还给软件**——用户全程只做 2 个创作决策,其余全部后台自动。

## 3. 目标用户与核心任务

- **用户:** 中文网文创作者 / 想"借鉴不抄袭"地快速起书的人。**非技术用户**,不接受任何参数术语。
- **核心任务:** 把一本已验证的爆款"骨架"换上新题材,拿到一段可用的新书开篇正文。

## 4. 核心理念:换皮变题(理论地基)

把小说拆成**可移植引擎**与**可替换皮**,迁移引擎、替换皮 → 新书。学理支撑:fabula/syuzhet(俄国形式主义)、Propp 功能序列"功能不变·角色可替换"、Riedl「story analogues」类推迁移。**这是合规的"借鉴",非抄袭。**

**DNA 4 层模型**(详见引擎蓝图 §2):

| 层 | 含义 | 来源 | 表示 |
|---|---|---|---|
| ① 结构骨架 | Propp 功能节拍序列 + 冲突引擎 | 骨架书 | 结构化(typed) |
| ② 编排节奏 | 视角/悬念铺陈/爽点曲线 | 骨架书 | 结构化 |
| ③ 题材皮 | 题材/世界观/意象 | 题材书 | 自由文本 |
| ④ 文笔 | 语感 | 默认每次贴新题材重生成 | 自由文本 |

## 5. 北极星用户旅程(全程仅 2 个创作决策)

```
配模型(一次性) → 选模式[自我裂变/交叉融合] → 导入 TXT
  → 自动切分(出问题才提示) → 后台自适应 DNA 提取(零参数,跑完通知)
  → 配方台:指认「哪本骨架 / 哪本题材」(+对调,+可选"想要什么"框)
  → 一键生成 3 个方向
  → 【决策① 三选一方向】
  → 后台:换皮迁移 + 补洞(修逻辑硬伤)
  → 确认设定台【决策② 确认/微调设定】
  → 流式开篇正文 → 复制/导出
```

## 6. 功能需求(FR)

- **FR1 模式选择:** 首屏两入口——自我裂变(1 本)/ 交叉融合(2 本)。
- **FR2 导入与切分:** 拖入 TXT,自动编码识别/清洗/切分;**仅在切分异常时**浮现修复入口,正常情况用户无感。
- **FR3 后台自适应提取:** 导入后**自动**在后台提取 DNA;**按体量自动路由**(小=整本单次/中=按弧层级/大=饱和采样收敛即停);用户**不接触任何参数**(无 100/全量、无档位);可离开,跑完通知"DNA 就绪"。失败章自动重试,中断自动续跑。
- **FR4 配方台:** 交叉融合时指认骨架/题材(含 ⇄ 对调);单本模式下题材来源=用户口述;一个**可留空**的"想往哪写/想避开什么"自然语言框。**无任何滑块/参数。**
- **FR5 生成方向:** 一键生成 **3 个**换皮嫁接方向(每个标注引擎/题材配方)。
- **FR6 决策①:** 用户三选一。
- **FR7 补洞(自动):** 选定方向后,后台对"新题材撑不起的结构节拍"定位并补入自洽事件——**质量护城河**。
- **FR8 确认设定台(决策②):** 展示换皮后**具体新书设定**(世界观/主角/对手/基调 等块)+ 每块只读**"引擎来源/题材来源"溯源标**。共创能力:
  - 手动直接改文字;
  - 一个 scope-aware 的 AI 框(对话/指令二合一);
  - **✨意图增强**:糙指令 → 先展示"我理解你要…对吗" → 确认后才执行(meta-prompt 带确认门);
  - **所有 AI 改动走 diff → 接受/拒绝/再调**(严禁静默覆盖);
  - **版本历史 + 一键回退**(试错零成本)。
- **FR9 成稿:** 流式生成开篇正文(墨落纸式显现);**轻量共创**(选中句 → AI 改写 → 接受/拒绝,不做整篇 diff);重写/续写/复制/导出 .md。
- **FR10 资料管理:** 书架(每本 DNA 状态)+ 创作库(已存创作,可重命名/删除)。

## 7. 非功能需求(NFR)

- **NFR1 重步骤异步后台化:** 提取/换皮/补洞/整本直提一律做成**异步后台任务**(发起→通知),不阻塞用户;以此规避 Vercel 10s 限制(见 §9 决策 A)。
- **NFR2 一切持久化于浏览器:** IndexedDB(Dexie),刷新/崩溃零丢失;断点续传;挂载自愈。**(现状已实现,保留)**
- **NFR3 BYOK:** 密钥永不落服务端;本地混淆存储;损坏密钥兜底为未配置。**(已实现,保留)**
- **NFR4 友好错误与限速护航:** 中文结构化错误;429 静默退避自动重试,不中断、不报红。**(已实现,保留)**
- **NFR5 后端安全防线:** SSRF allowlist + 按 IP 限速 + 密钥脱敏日志。**(已实现,保留)**
- **NFR6 资源友好:** 大文件分块读取;16GB RAM 为真实上限,避免一次性巨量驻留。

## 8. UX 契约(不可协商)

> **`docs/redesign-mockup.html` 是用户操作体验的唯一真相源。任何前端实现以它为准;偏离原型的布局、流程或交互即视为不通过验收。**

- **视觉系统「朱墨」:** 暖墨黑 + 纸张米 + 唯一朱砂红(#cf4a2e)强调;字体 Fraunces + 思源宋体 + JetBrains Mono;**骨架/皮以"材质二元"呈现**(冷蓝图等宽 vs 暖纸纹宋体)。
- **动画 = 仅功能性:** 保留 正文流式(墨落纸)、diff 高亮、极简进度、当前步高亮;**删除一切装饰**(星体碰撞、双螺旋、3D 翻卡、Genomics Live Feed、发光渐变、青色脉冲)。

## 9. 三个锁定决策

- **A 部署/长任务:** 重 LLM 步骤**异步后台化**为最佳实践(非"硬撑长请求");"长超时/非 Vercel 后端"为兜底下限。**不为 10s 限制牺牲质量。**
- **B 存量 DNA 卡:** 旧 5 维卡**标记 legacy + 原文不丢**,重新提取时才升级 4 层(惰性、非破坏式迁移,与仓库既有惯例一致)。
- **C 设定台编辑对象:** 编辑**换皮后具体新书设定 + 溯源标**(沿用现有块形状,改动最小,仍呈现引擎/皮二元);不直接编辑抽象 4 层。

## 10. 不做 / 删除(Non-goals & Kill-list)

档位(safe/balanced/speed)、100/全量切换、fusionBias 偏向滑块、temperature 滑块、单列红队约束框(反套路改后台常开)、手动分镜数、手动"生成故事板"独立步骤、命令栏"自动命中+青色脉冲"、前台双螺旋盯等、所有装饰动画。**反套路约束(ANTI_SLOP)在后端始终生效。**

## 11. 验收标准(AC)

1. **交付 UX 逐屏符合 `redesign-mockup.html`**(binding)。
2. 用户全程**创作决策 ≤ 2**(三选一 + 确认设定)。
3. 用户**全程不接触任何参数旋钮**。
4. 小/中/大三种体量都能自动提取:中短篇一次过、《斗破》级**不卡死、不偏科**。
5. 换皮产出**经补洞**,开篇无明显逻辑断裂。
6. 任何 AI 改动**先 diff、可拒、可回退**,无静默覆盖。
7. 刷新/崩溃**零数据丢失**,提取可续跑。

## 12. 里程碑(详见引擎蓝图 §7)

Phase 0 数据/schema 地基 → 1 自适应提取引擎 → 2 换皮+补洞 → 3 共创(diff/版本/意图增强)→ 4 后台化与清理 → **5 = UI 落地(逐屏对齐原型)**。每阶段 `tsc + lint + build` + 手动走查;无自动测试套件。

## 13. 风险

- 重步骤超 Vercel 10s(→ 异步后台化 / 非 Vercel 后端)。
- 4 层 DNA 重切触及双侧 schema 同步(schemas.py ↔ db.ts/组件),须逐字段对齐。
- 换皮+补洞为全新管线,prompt 质量需迭代调优。
- 改动面大,须严守"对账真实代码、不照理想态硬写"(见记忆 [[bmad-epics-idealized-vs-shipped-code]])。
